§1The sleeper token

Take a sentence whose subject only resolves at the end — "the crane lifted its long neck over the marsh" — and watch a single token's hidden state across the stack. At the input it's just a string. Somewhere mid-stack it commits to bird, not machine, and you can see exactly when.

The reading direction matches the computation: tokens across the top, depth increasing downward, so each column reads as a token falling through the stack. Identical regions recede to the page ground; divergence ignites. The single injected/cause column ignites citron, so cause and effect read as warm-versus-cool inside one chart.

The question Not "can the model do this," but "when in the stack does it know?" Access and computation turn out to be two independently gated things — which is the whole reason a scan beats a score here.
Commitment lags the cue. The model knows the answer several layers before it is willing to say it. The day-zero scan · §1 · read against a position-shuffle null

§2Two gates, opening independently

The long-range staircase separates access (can the model reach back to a distant token at all) from computation (does it do the depth-work once it has). They're gated at different layers — which means "it failed the task" is at least two different failures, and the scan tells you which one.

Known limitations Confidencelow
  • Single base model — no cross-architecture replication yet; "how transformers are" is a hypothesis, not a result.
  • Geometry read against a position-shuffle null only, not a baseline-model null. "Position-robust" ≠ "structure-sensitive."

Colophon & method

Hidden states pulled via class-level __call__ hooks, bf16→f32 cast, centred cosine, difflib token alignment, last-token vs mean-pool both recorded. Built and run locally on a 16 GB Mac Mini. The probe method owes its origin to David Ng's RYS line; the geometry read is mine. Full instrument details → Methods.
