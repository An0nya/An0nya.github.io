Most of these experiments end in a characterised breakage rather than a better model, and that's the point: the negative and corrected results are kept on purpose, with the autopsy.

How I work
In one line

The differentiator isn't a model that got better - it's the measurement layer underneath: seed discipline, paired effect sizes, and a strict regrade that keeps catching the pretty results lying. I direct and decide; large models read code, run the statistics, and draft. The judgment about what's true is mine.

Measured, not marketed

Every number traces to a file in a repo - config hash in the filename, raw per-probe JSONL, strict score leading. No comparison that isn't backed by an artifact.

Negatives are first-class

Two headline findings here were wrong and stayed on the page with the autopsy. A notebook that only records confirmed results is a brochure.

Lean by conviction

Static HTML, ~1 KB of JS, hand-authored SVG figures, no framework. Greppable all the way down — and it loads instantly on a phone.
